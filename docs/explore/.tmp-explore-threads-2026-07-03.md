## 线索 1 深挖：架构 PRD 自洽性（2026-07-03）

### 总评

**主干设计是自洽的**：Manager / Data 分离、PostgREST 单一 CRUD、Opaque Token、双路径 project 解析（Session vs URL）在 §6–§11 反复出现且大体一致。

**主要问题不在「互相打架」，而在**：(1) 个别段落写错或过度简化；(2) **两套权限模型**（平台成员 vs 租户用户）边界须写清 — ✅ §11.3 / §11.11 已统一 SET ROLE；(3) ~~RLS 目标态 vs MVP 网关策略~~ ✅ 已合并为 bootstrap RLS + db-pre-request。

### 边界图（PRD 意图）

```
┌─────────────────────────────────────────────────────────────────┐
│                        Axum 单进程                               │
├────────────────────────────┬────────────────────────────────────┤
│ Manager API                │ Data API 网关                       │
│ /api/projects, /tables…    │ Dashboard: /api/data/*              │
│ /api/auth/*                │ SDK:       /api/data/{project_id}/* │
├────────────────────────────┼────────────────────────────────────┤
│ 凭证：Dashboard Session    │ Dashboard: Project Session          │
│       或 Secret Key (S2S)  │ SDK: Publishable Key (+ App User)   │
│                            │      或 Secret Key (admin bypass)   │
├────────────────────────────┼────────────────────────────────────┤
│ sqlx 直连 public           │ 代理 → PostgREST → proj_{ulid}      │
│ DDL / Metadata / Storage   │ Axum 不实现 CRUD                    │
└────────────────────────────┴────────────────────────────────────┘
         │                              │
         └──────── Redis ──────────────┘
              (三类 Session + Key cache)
```
